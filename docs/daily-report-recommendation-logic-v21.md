# 五行投资日报推荐逻辑说明 V2.1

本文档描述当前线上代码的真实推荐逻辑，覆盖整体策略、行业推荐、个股推荐、风险控制和已知限制。

## 1. 目标和输出定位

五行投资日报不是直接给出“单一买点”，而是把当日市场信息压缩成三个层级：

1. 整体策略：判断当天偏进攻、持仓观察、中性观察或防守降噪，并给出仓位节奏。
2. 行业候选池：用申万31行业五行暴露 V2.1 生成行业排序，重点输出 Top3-Top5。
3. 个股关注池：从强势行业候选标的中筛 3-6 个关注对象，作为后续人工或规则确认入口。

当前回测结论也支持这个定位：算法更适合作为“Top5 行业候选池/方向过滤”，不适合作为“Top1 重仓买入信号”。

## 2. 数据流

计划任务在 `09:25 / 11:30 / 13:00 / 15:10` 触发 `bin/daily-auto-generate.js`。

主要输入：

- A股指数：沪深300、上证、深成指、创业板。
- 市场宽度：上涨家数、下跌家数、平盘家数。
- 情绪温度：涨停、跌停。
- 资金动量：北向资金、主力资金、ETF 流向、换手率。
- 海外市场：道指、纳指、标普、恒生。
- 五行日历：四柱、五行强弱、年度流年修正。
- 行业框架：申万31行业五行暴露 V2.1。

输出落库：

- `reports`：日报主记录。
- `stocks`：关注标的。
- `decision_logs`：日报结论、关键变量、推荐行业和市场结果。
- 前端读取 `/api/latest` 或 `/api/reports/:date` 展示。

## 3. 整体策略逻辑

整体策略由 `buildOperationAdvice()` 生成，核心依据是指数状态、创业板弹性、海外风险和资金动量。

### 3.1 AND / OR 关系

当前逻辑是“分层门槛”，不是所有条件简单相加。

偏进攻是严格 AND：

```text
沪深300 > +0.8%
AND 创业板 > +1%
AND 纳指 >= 0
AND 主力资金净流入或不为负
```

含义：进攻必须同时满足指数、成长风格、外盘风险偏好和资金承接四个条件，避免单一指数上涨就扩大仓位。

持仓观察是 AND + OR：

```text
沪深300 > 0
AND 上证 > 0
AND (主力资金 >= 0 OR 北向资金 >= 0)
```

含义：市场本身要站在正收益区间，同时至少有一类资金没有转弱，才允许维持仓位。

防守降噪是 OR：

```text
沪深300 < -1%
OR 恒生指数 < -1%
```

含义：防守按风险优先原则处理，只要核心宽基或外部风险显著恶化，就先降低风险暴露。

如果以上条件都不满足，则为中性观察。

这种设计背后的投资依据是“不对称风控”：进攻需要多因子共振，防守只需要单个关键风险触发。这样宁可少做一点，也不让单一乐观信号把仓位推高。

策略状态：

- 偏进攻：
  - 沪深300涨幅大于 0.8%。
  - 创业板涨幅大于 1%。
  - 纳指不弱。
  - 主力资金净流入。
  - 建议：稳健账户 5-6 成，激进账户不超过 7 成。

- 持仓观察：
  - 沪深300和上证为正。
  - 主力资金或北向资金至少一项不弱。
  - 建议：维持 4-5 成，保留趋势仍在的仓位。

- 防守降噪：
  - 沪深300跌幅超过 1%，或恒生跌幅超过 1%。
  - 建议：降到 3 成以内，先处理弱势标的。

- 中性观察：
  - 以上条件均不满足。
  - 建议：稳健仓位控制在 4 成以内，等待量能和主线确认。

风险文案由 `buildRiskScenario()` 生成，明确区分事实、预测和风险场景。例如指数已上涨时，不写“会跌1%”，而写“当前上涨，但模型提示从盘中高位回撤/涨幅收窄风险”。

## 4. 五行和年度修正

四柱五行强弱由 `countFiveElements()` 计算：

- 天干：1 分。
- 地支本气：0.5 分。
- 地支藏干：0.25 分。

年度修正来自 `backend/swIndustryFramework.js`：

- 2026 丙午：火 +25%、木 +5%、土 +5%、水 -10%、金 -15%。
- 2027 丁未：火 +15%、土 +10%、木 +5%、水 -5%、金 -5%。
- 2028 戊申：金 +15%、水 +5%、土 +5%、木 -5%、火 -10%。
- 2029 己酉：金 +20%、水 +5%、土 +5%、木 -10%、火 -15%。
- 2030 庚戌：金 +15%、土 +10%、水 +5%、木 -5%、火 -5%。

年度修正会作用在行业五行暴露和当日五行信号上。

### 4.1 东方玄学理论如何进入投资模型

当前系统没有把玄学当作“直接预测涨跌的神秘因果”，而是把东方术数转译成一套行业因子标签和风险语言：

- 月令：代表当下大环境的主气，类似市场风格背景。
- 日干/日支：代表当日能量结构，类似短周期信号。
- 五行强弱：转成行业暴露偏好，木火土金水分别对应不同产业属性。
- 喜用/忌神：转成增配/降权方向。
- 生克制化：转成行业之间的顺势、受压、泄耗关系。
- 流年：转成年度行业因子修正，例如 2026 丙午火旺，对火性行业加权，对金水行业压制。
- 十神：当前只作为辅助确认，不覆盖申万31行业主排序。

换句话说，玄学部分提供“结构化先验”，最终是否进入推荐仍要被指数、资金、行业强度和回测结果验证。

## 5. 行业推荐逻辑

主排序函数：`getEnhancedIndustries()`。

底层评分函数：`scoreSwIndustries()`。

当前可执行评分流程：

1. 读取申万31行业基础五行暴露表。
2. 对每个行业应用年度流年修正。
3. 将修正后的行业五行暴露做 Softmax 归一化。
4. 计算当日五行信号，包括五行强弱、喜用元素、主导元素、年度修正。
5. 用“行业暴露 x 当日五行信号”点积得到基础分。
6. 如果涨停/强势股票名称命中行业关键词，该行业加 `0.04` 市场确认分。
7. 所有行业按分数排序，再归一化为 `factor_score` 0-100。
8. 按排名分位给星级：
   - 前 15%：5 星。
   - 前 35%：4 星。
   - 后 20%：2 星。
   - 其他：3 星。

当前场景权重：

- 中短线投资：政策 30% + 资金动量 30% + 周期 20% + 价值链 10% + 要素 10%。

注意：这组场景权重当前主要作为框架说明和展示元数据；可执行评分尚未把政策、资金动量、周期、价值链、要素拆成独立数值子模型。现阶段真正参与排序的是“申万31行业五行暴露 + 当日五行信号 + 年度修正 + 盘中强势行业确认”。后续权重工程化应把这些维度拆成可回测的独立因子。

## 6. 行业压力测试

每个行业会生成压力测试字段：

- 年度修正压制主导五行。
- 沪深300跌幅超过 1%。
- 主力资金净流出。
- 北向资金净流出。
- 火因子行业在指数大涨后存在回撤风险。
- 2026 火旺克金时，金性行业降权观察。

风险等级：

- `normal`：没有明显触发。
- `elevated`：至少 1 条风险。
- `high`：至少 3 条风险。

前端行业卡片会展示关键变量和压力测试。

## 7. 个股推荐逻辑

个股推荐分为自动日报关注池和单只股票分析两部分。

自动日报关注池：

1. 取行业排序前 6 名。
2. 为每个行业带出候选标的，来源是代码里的行业-标的映射。
3. 优先调用 LLM，让模型在候选标的中选 3-5 只，并要求只输出 JSON。
4. LLM 失败时，退回静态规则：
   - 取 4 星以上行业的第一候选标的。
   - 再从 Top6 行业候选标的中补足最多 6 只。
5. 若整体策略是“防守降噪”，补充标的建议为“观望”；否则为“关注”。

当前 LLM 只负责“从候选里写成关注标的和理由”，不改变行业主排序。若 LLM token 不可用，系统会自动使用静态推荐。

个股策略的实际含义：

- 先选行业，再选个股；不做脱离行业的单票推荐。
- ETF 和龙头股优先，因为它们更能代表行业 beta，和当前行业回测口径一致。
- 防守状态下，个股建议自动偏向“观望”，不会因为行业分高就鼓励追买。
- 个股只进入“关注池”，不是自动买入池；最终还要看盘口承接、成交额、资金流和个股自身风险。

结合东方玄学的表达方式：

- 行业是“气”的方向，个股是“器”的承载。
- 五行强、流年顺、资金确认，说明行业之气较顺；个股还必须量价和资金承接配合，才算“气有所载”。
- 五行受克、流年压制、资金流出，即使个股短线异动，也只作为观察，不作为主动追高依据。

单只股票分析：

接口：`GET /api/stock/analyze?q=股票代码或名称`。

它基于实时行情做四维规则分析：

- 消息/情绪：根据涨跌幅判断亢奋、回调、横盘等。
- 基本面：PE/PB 与估值提示。
- 技术面：根据涨跌幅粗略判断强势突破、回调、横盘。
- 资金面：根据主力净流入/净流出提示承接或抛压。

这部分不是深度个股模型，不能替代财报、公告、成交量结构和人工风控。

## 8. 回测验证口径

当前已完成两类回测：

1. 投资者视角回测：
   - 验证 Top1/Top3/Top5 行业篮子对投资者是否有帮助。
   - 结论：Top5 行业候选池有效性优于 Top1 集中押注。

2. Alpha 诊断：
   - 验证排序是否有 alpha。
   - 指标包括 Top5 平均超额、Top5-全行业提升、Top5-Bottom5 分层、Rank IC、未来最强行业捕获率。

当前日线回测口径：

- 收盘后得到信号。
- 下一交易日开盘买入。
- 持有 1/3/5/10 个交易日。
- 相对沪深300计算超额。

重要限制：早盘和午间日报需要用当时可见的盘中数据重新回测，不能用收盘后数据证明盘中决策有效。

## 9. 当前产品建议

- 日报文案应强调“Top5 行业候选池”，不强调“第一行业必买”。
- 组合建议默认分散到 3-5 个行业。
- Top1 只作为关注度排序，不作为仓位排序。
- 对高频轮动要显式展示交易成本和滑点。
- 后续优化重点是把政策、资金动量、周期、价值链、要素拆成独立因子，并通过 `decision_logs` 做持续校准。
