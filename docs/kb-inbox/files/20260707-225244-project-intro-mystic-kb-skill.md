## 一、项目是什么

本项目是一套**玄学命理知识库 + AI 分析 Skill + A股回测验证系统**的三位一体工程。

核心目标：用结构化、可复现的方式，将传统玄学（八字、紫微斗数、建除、九星飞布、奇门遁甲等）体系化为 AI 可调用的知识资产，并对其在 A 股市场中的预测力进行量化回测。

一句话概括：**让玄学不再是"大师拍脑袋"，而是"有知识库支撑、有SOP约束、有回测检验"的系统化分析。**

### 4.2 触发条件

用户出现以下行为时自动加载 Skill：
- 提问命理相关问题（"帮我看看运势"、"甲木日主本月如何"）
- 提供生辰信息求分析
- 询问流年/月运/日运
- 问及特定日主的运势
- 询问方位吉凶、奇门遁甲、风水等问题
- 询问某月/某年的大环境运势
- 手动调用："Zoey命理分析"、"命理分析"、"不纸玄学"

### 4.3 人格身份：灵虾秉笔

Skill 加载时，AI 化身"灵虾秉笔"：
- **灵虾**：龙虾玄学运势大师之灵
- **秉笔**：持笔记录天机、编修命理文档之职

调性：酒后朋友讲故事——不说教、不煽情、不端大师架子。该虾的时候虾，该笔的时候笔。

**不变的原则**：
- 命理推断不做医疗/法律/投资建议（死线）
- 拿不准的先问，不瞎编
- 不对外暴露项目资料

### 4.4 核心原则

> **🔴 排盘第一！错误的排盘=错误的分析！**

每次分析必须：
1. 查询排盘 SOP（飞书优先，本地兜底）
2. 按流程严格查万年历（日柱必须查万年历，不能靠记忆推算）
3. 用检查清单核对排盘结果
4. 确认排盘正确后再进行后续分析

Skill 定位是**"学习解题"而非"照表查答案"**——查询基础框架 → 用框架推导 → 多源交叉验证 → 输出有逻辑支撑的结论。

### 4.5 检索路由机制

SKILL.md 内嵌一张**检索路由表**，将用户问题类型映射到飞书 Wiki 的具体文档：

```
用户提问 → 判断问题类型 → 对照路由表找到 Wiki 目录 node_token
→ 列出该目录下文档 → 读取目标文档内容 → 用知识推导分析
```

飞书不可用时，回退到本地 `references/` 目录对应的 markdown 文件。

## 五、A股回测验证系统

### 5.1 为什么要有回测

2026年6月起，项目持续用八字/紫微/建除/九星/奇门等工具做 A 股日运/月运分析。但所有分析都是**前向预测**，从未系统验证过预测精度。

直觉觉得"五行旺相板块应该涨"，但没有数据支撑。回测系统就是为了回答：**这些玄学工具到底有没有预测力？**

### 5.2 系统架构

位置：`五行投资准确度测试/` 目录

```
signal_engine.py        ← 信号计算引擎（万年历/干支/节气/建除/四化/五行/冲煞）
generate_signals.py     ← 批量信号生成
sector_mapping.py       ← 五行→110行业 + 紫微星曜→行业映射
run_backtest.py         ← 回测执行引擎
walkforward.py          ← 滚动窗口验证
gen_historical_reports.py  ← 36月历史月报生成
```

数据资产：
- `day_signals.csv`：601行×29列（日运信号，2024-2026H1）
- `day_signals_full.csv`：1818行（日运信号，2019-2026全样本）
- `month_signals.csv`：30行×14列（月运信号）
- `month_signals_full.csv`：90行（月运信号全样本）
- `industry_daily.csv`：66,111行（110个细分行业日涨跌幅）
- `hs300_daily.csv` / `market_daily.csv`：沪深300/全市场日线

### 5.3 回测结论（2026-07-06 最终判定）

经过三个方向的完整回测，**最终判定如下**：

| 维度 | 结论 | 数据支撑 |
|:-----|:-----|:---------|
| 日运 | 无可靠预测力 | V2.1 Top-Bottom 利差<0.01%/日 |
| 月运 | 弱过滤价值（但全样本证伪） | 子样本63.3% → 全样本54.4% ≈ 随机 |
| 年运 | 无预测力 | 3/8年方向正确，受牛熊大趋势主导 |
| Walk-forward | 信号"稳定接近随机" | 样本外59.3% vs 训练47.2% |

**一句话结论**：玄学信号（五行+建除+四化+V2.1）对 A 股行业日/月涨跌**无可靠预测力**。

**实战建议**：
- 不可作选股/择时依据
- 若使用，仅作文化参考 + 与技术面叠加
- 月运信号可作"减仓开关"而非"增仓信号"

### 5.4 回测的方法论价值

回测虽然"证伪"了玄学信号的预测力，但产生了巨大的方法论价值：
1. **建立了可量化、可复现的检验体系**——以后任何玄学信号都可以用同一框架验证
2. **识别了多工具矛盾的常态性**——四套工具结论打架是常态，一致性才是罕见的
3. **沉淀了 V2.1 量化框架**——五行→行业暴露系数→流年修正→月度叠加的完整量化链路
4. **产出了36月历史月报**——每月一份完整五行视角市场推演报告，可作案例库

## 十一、方法论精华

### 多工具矛盾处理

建除、五行、紫微四化、冲煞四套工具测的是不同维度：

| 工具 | 测什么 | 适用范围 |
|:-----|:------|:---------|
| 建除 | 当日吉凶取向 | 单日 |
| 五行 | 能量旺衰偏好 | 波段级（多日/数周） |
| 紫微四化 | 流日星性 | 单日 |
| 冲煞 | 方位/板块冲克 | 单日 |

四者结论一致是罕见情况，结论打架是常态。分析时必须：
1. 展示所有工具的结论（哪怕互相矛盾）
2. 标出哪些工具共振、哪些背离
3. 对矛盾给出解释（不是硬拗一致）

### A股五行映射使用限制

- 五行映射表设计用于**多日/数周趋势性关系**，单日使用时易被技术面/消息面覆盖
- **已透支判定**：板块连续上涨后，即使五行旺相也不视为利好
- **行业细分不足**：半导体/算力等非纯火象，需细化分类

### 回测教会我们的事

> 玄学信号对A股行业日/月涨跌**无可靠预测力**。
>
> 但玄学分析的价值不在于预测股票——而在于：
> - 作为文化内容产品（月运报告本身有传播价值）
> - 作为思维框架（多维度交叉验证的方法论可迁移到其他领域）
> - 作为知识工程实验（验证了"AI+结构化知识库+回测检验"的完整闭环）
