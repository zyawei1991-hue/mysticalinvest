## 结论

日报要从算法展示变成面向用户的操作翻译器。

第一屏优先回答：

1. 今天能不能进攻。
2. 如果要做，做什么方向。
3. 什么人不适合做。

## 用户类型

| 用户 | 输出重点 |
| --- | --- |
| ETF 用户 | 行业 ETF、宽基 ETF、仓位建议 |
| 个股用户 | 候选池，不给必买结论 |
| 已持仓用户 | 持有、减仓、换仓、观察 |
| 稳健用户 | 风险提示、少交易、等待确认 |
| 玄学兴趣用户 | 五行逻辑放在依据和附录，不抢主结论 |

## 用在日报里

- 先给动作，再给依据。
- 先讲行业和 ETF，再讲个股候选。
- 明确“不适合做”的人群和场景。

## 禁止用法

- 不用大段玄学解释压过主结论。
- 不把候选池写成确定性买入清单。
