投资日报子知识库文件工作流

当前阶段按“团队定期发文件”的方式建设投资日报子知识库。所有文件先进入草稿和审核，不直接影响日报程序。

1. 发文件Markdown、HTML、CSV、XLSX、导出件
2. inbox保存原始材料副本
3. draft自动生成待审核知识卡
4. review人工确认边界和用途
5. active进入日报程序可调用知识库

当前目录

目录/文件作用
docs/kb-inbox/files/保存团队定期发来的原始文件副本。
docs/kb-drafts/保存自动生成的 draft 知识卡。
docs/kb-change-requests/保存对 active 知识的修改提案。
docs/kb-templates/保存知识卡模板。
docs/kb/正式知识库源文件，只放审核后的知识。
data/kb-audit-log.jsonl记录导入、发布、修改等审计事件。

文件导入

当团队在群里定期发来文件后，先保存到 attachments/，再执行导入命令。

node bin/import-kb-file.js --source attachments/example.md --title "材料标题" --domain strategy

生成结果：

  • 原始副本：docs/kb-inbox/files/
  • 草稿卡片：docs/kb-drafts/
  • 审计记录：data/kb-audit-log.jsonl

导入后的 draft 不会被日报程序默认读取。

人工审核

审核时优先判断这份材料属于哪个层级：

  1. 母知识库理论：应回写母库，不直接进入投资日报。
  2. 投资日报映射：可进入 market / strategy。
  3. 输出规范：可进入 output。
  4. 合规边界：可进入 compliance。
  5. 案例复盘：可进入 case。

审核必须补齐：用途、禁用用法、来源、信心等级、是否需要回测。

发布 active

审核通过后，把 draft 发布进正式知识库并重建索引。

node bin/promote-kb-draft.js --draft docs/kb-drafts/example.md --domain strategy --reviewer owner-name
node bin/build-knowledge-base.js

发布后可通过页面查看：

  • http://117.72.58.55/daily/knowledge-base.html
  • http://117.72.58.55/daily/knowledge-base-data.json
  • http://117.72.58.55/daily/api/knowledge-base?q=关键词&usable_for=daily_report

禁止规则

  • 不把原始文件直接作为日报程序依据。
  • 不把 draft 草稿直接发布到 active。
  • 不把未验证的玄学信号写成投资建议。
  • 不删除审计日志。
  • 不物理删除 active 知识，废弃时标记 deprecated。
